基于 [AstrBot](https://github.com/Soulter/AstrBot) 的群聊二次元老婆互动插件。每日随机抽取二次元老婆，支持牛老婆、换老婆、交换老婆、要本子，以及完整的群友共建老婆库流程（搜索 → 审核 → GitHub PR → 上线）。

## ✨ 功能亮点

- **每日老婆**：每人每日随机抽取一名二次元老婆，重复抽取返回当日结果。
- **牛老婆**：概率性抢走他人今日老婆，有失败风险。
- **换老婆**：放弃当前老婆并立即重抽，每日次数有限。
- **交换老婆**：向其他群友发起双向交换请求，需对方同意。
- **要本子**：基于当日老婆角色，联动搜索 JM / NH / EH / DL 等资源。
- **共建老婆库**：群友可搜索角色提交申请，管理员审核后通过 GitHub PR 自动上线。
- **邓哥曾姐惩罚**：多次重置换老婆后，下次抽老婆有概率触发惩罚图（概率随次数累加）。
- **图片压缩传输**：自动将老婆图片压缩至 400KB 以内再发送，避免 LLOB 拉取超时。

## 🎮 使用指令

### 基础功能

| 指令 | 权限 | 说明 |
| --- | --- | --- |
| `抽老婆` | 用户 | 随机抽取今日二次元老婆 |
| `查老婆 [@用户]` | 用户 | 查看自己或他人今日老婆 |
| `要本子` | 用户 | 根据今日老婆搜索相关本子资源 |
| `老婆帮助` | 用户 | 显示所有指令说明 |

### 牛老婆

| 指令 | 权限 | 说明 |
| --- | --- | --- |
| `牛老婆 @用户` | 用户 | 尝试抢走对方今日老婆（有概率失败） |
| `重置牛 [@用户]` | 用户 | 消耗重置次数，有概率重置自己/对方的牛次数（失败会禁言） |

> 管理员执行 `重置牛 @用户` 可直接重置，不消耗次数、不触发禁言。

### 换老婆

| 指令 | 权限 | 说明 |
| --- | --- | --- |
| `换老婆` | 用户 | 放弃今日老婆并立即重抽，每日有次数上限 |
| `重置换 [@用户]` | 用户 | 消耗重置次数，有概率重置自己/对方的换老婆次数（失败会禁言） |

> 管理员执行 `重置换 @用户` 可直接重置。
> 他人帮你成功重置换次数后，你下次 `抽老婆` 触发邓哥惩罚的概率会增加。

### 交换老婆

| 指令 | 权限 | 说明 |
| --- | --- | --- |
| `交换老婆 @用户` | 用户 | 向指定用户发起老婆交换请求 |
| `同意交换 @发起者` | 用户 | 同意对方的交换请求，双方老婆互换 |
| `拒绝交换 @发起者` | 用户 | 拒绝对方的交换请求 |
| `查看交换请求` | 用户 | 查看当前发出和收到的交换请求 |

### 添老婆

| 指令 | 权限 | 说明 |
| --- | --- | --- |
| `添老婆 角色名/作品名` | 用户 | 搜索角色候选并提交审核申请 |
| `补充来源 作品名` | 用户 | 补充最近一次提交中缺失的作品来源 |
| `刷新缓存` | 管理员 | 强制重建英文名翻译缓存（调用 AI 翻译，速度较慢） |

### 管理员（NTR）

| 指令 | 权限 | 说明 |
| --- | --- | --- |
| `切换ntr开关状态` | 管理员 | 开启或关闭本群的牛老婆功能 |

## 📬 添老婆流程

群友可以将自己喜欢的角色提交到老婆库，流程如下：

```
群友：添老婆 角色名/作品名
  ↓ 插件搜索 Bangumi + AniList，展示最多 3 个候选及预览图
群友：回复数字选择（支持「换一批」翻页，「取消」退出）
  ↓ 提交至待审核队列，私聊通知管理员
管理员（私聊）：通过 <序号>  /  拒绝 <序号>
  ↓ 通过后插件自动拉取角色图片，发给管理员确认
管理员（私聊）：确认 <pid>  /  换图 <pid>  /  跳过 <pid>
  ↓ 确认后自动创建 GitHub PR，更新 list.txt
管理员 merge PR 后：pr上线 <pid>
  ↓ @通知提交者「角色已上线，快去抽老婆试试！」
```

## 🔧 管理员功能

所有管理员操作均通过**私聊 Bot** 完成：

| 指令 | 说明 |
| --- | --- |
| `拉取老婆审核` | 列出所有待审核申请，附序号 |
| `通过 <序号>` | 通过申请并自动拉取图片 |
| `通过 <序号> 作品名` | 通过的同时补充来源 |
| `拒绝 <序号>` | 拒绝申请并通知群友 |
| `确认 <pid>` | 确认图片并创建 GitHub PR |
| `换图 <pid>` | 重新拉取一批图片 |
| `跳过 <pid>` | 跳过图片搜索，创建空 PR（手动上传图片后 merge） |
| `pr上线 <序号/pid>` | 标记上线并 @通知提交者 |

## 🛠️ 安装与配置

### 依赖

```
Pillow
aiohttp
```

安装插件后会自动创建数据目录：`data/astrbot_plugin_animewifex/`

### GitHub 图床仓库格式

老婆图片存储在 GitHub 仓库中，文件名格式为：

```
作品名!角色名.jpg
```

例如：`魔法少女小圆!鹿目圆.jpg`、`eden*!紫苑.png`

仓库根目录需有 `list.txt`，每行一个文件路径，插件每小时自动拉取缓存。

## ⚙️ 配置项说明

在 AstrBot 管理面板或 `_conf_schema.json` 中配置：

### 基础配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `need_prefix` | bool | false | 是否需要 @Bot 或唤醒词才能触发指令 |
| `image_base_url` | string | — | 图床基础 URL，图片路径会拼接在此后 |
| `image_list_url` | string | — | `list.txt` 的直链地址，插件用于随机抽取老婆 |
| `admin_qq` | string | — | 管理员 QQ 号，审核通知将私聊发送到此账号 |

### 功能限制配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `ntr_max` | int | 3 | 每人每日牛老婆次数上限 |
| `ntr_possibility` | float | 0.3 | 牛老婆成功概率（0~1） |
| `change_max_per_day` | int | 3 | 每人每日换老婆次数上限 |
| `swap_max_per_day` | int | 3 | 每人每日发起交换请求次数上限 |
| `reset_max_uses_per_day` | int | 2 | 每人每日重置次数上限（牛/换共用） |
| `reset_success_rate` | float | 0.5 | 重置成功概率（失败则禁言） |
| `reset_mute_duration` | int | 60 | 重置失败禁言秒数 |

### 添老婆配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `github_token` | string | — | GitHub Personal Access Token，用于创建 PR |
| `github_repo` | string | — | 图床仓库，格式：`用户名/仓库名` |
| `github_branch` | string | `main` | 目标分支名 |
| `pixiv_refresh_token` | string | — | Pixiv refresh token，用于搜索角色图片（可选） |

### XX惩罚配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `XX_img` | string | — | XX图片路径（`list.txt` 中的相对路径），留空则不启用 |
| `XX_img` | string | — | XX图片路径，留空则不启用 |
| `dengge_base_prob` | float | 0.15 | 每次被重置换成功后增加的触发概率 |
| `dengge_max_prob` | float | 0.80 | 惩罚概率上限 |
